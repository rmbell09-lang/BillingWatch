Self-Hosted Stripe Billing Monitor: Stop Paying $99/mo for What You Can Run Free

open-source stripe self-hosted April 3, 2026 · 6 min read

Baremetrics starts at $99/month. ProfitWell (now Paddle) has been absorbing features while the pricing bloats. For indie developers and small SaaS teams, this is a lot to pay just to know if your revenue is behaving normally.

BillingWatch does the core job for free. It is open-source, self-hosted, and runs on a $5/mo VPS. You get Stripe webhook processing and anomaly detection without handing your billing data to a third party.

What BillingWatch Does

BillingWatch is a Python application that:

  • Receives Stripe webhooks and processes them in real-time
  • Runs anomaly detection on your payment stream — surfacing spikes, drops, and unusual patterns
  • Stores event history locally — your data stays on your server
  • Alerts you when something looks wrong

It will not replace a full analytics suite. But if you need a sanity check on your billing — something that says "revenue dropped 40% in the last hour and here is why" — BillingWatch does that without the SaaS bill.

Quick Install

You need Python 3.8+, a Stripe account, and a server (or localhost for testing).

# Clone and install
git clone https://github.com/rmbell09-lang/billingwatch
cd billingwatch
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env: add STRIPE_WEBHOOK_SECRET and STRIPE_API_KEY

# Start the server
python app.py

Then add your BillingWatch webhook URL to Stripe:

  1. Go to Stripe Dashboard → Developers → Webhooks
  2. Click Add endpoint
  3. Enter your server URL: https://your-server.com/stripe-webhook/
  4. Select events: payment_intent.succeeded, invoice.payment_failed, customer.subscription.deleted
  5. Copy the signing secret into your .env as STRIPE_WEBHOOK_SECRET

That is it. Stripe starts sending events, BillingWatch processes them.

Anomaly Detection: How It Works

BillingWatch maintains a rolling baseline of your payment activity. When incoming events deviate significantly from that baseline — failed payment rate spikes, revenue drops, unusual charge patterns — it flags them.

This catches real problems: webhook delivery failures, Stripe configuration mistakes, payment processor outages, and the kind of subtle revenue bleed that is easy to miss when you are not watching a dashboard all day.

Self-Hosted Means You Own Your Data

Every Baremetrics integration means your Stripe API key and full billing history sit on someone else's servers. For most teams this is fine. But if you are handling sensitive subscription data, operating in a regulated industry, or just prefer to keep things in-house, self-hosting is not optional — it is required.

BillingWatch stores everything locally. No external API calls except back to Stripe for verification. Your billing data does not leave your infrastructure.
